# Plan: Create `/research` Claude Code Skill

## Context

The ClawFriend Skill Market project needs a **Claude Code skill** (`.claude/skills/` format) for market research. The user wants a `/research` slash command that fetches live competitor data, market trends, and updates the project's markdown deliverables. This is a Claude Code IDE skill — NOT a ClawFriend marketplace skill.

## Deliverables (4 files)

### 1. `.claude/skills/research/SKILL.md` — Main skill definition

YAML frontmatter + markdown body with:

```yaml
---
name: research
description: Market research for ClawFriend Skill Market — competitors, trends, on-chain data. Subcommands: competitors, market, skill <topic>, update.
---
```

**Subcommand routing via `$ARGUMENTS`:**

| Command | Action |
|---------|--------|
| `/research competitors` | Fetch competitor data (Virtuals, MyShell, ElizaOS, etc.) via CoinGecko/DefiLlama APIs + WebSearch. Update `competitive-landscape.md` |
| `/research market` | Market trends, BNB Chain TVL, AI agent market cap. Update `strategic-analysis.md` |
| `/research skill <topic>` | Deep-dive a skill category (defi, token, social, content, arbitrage). Update `skill-research.md` |
| `/research update` | Full refresh of all deliverables |

**Body content includes:**
- Pre-research step: read `data/README.md` for last-known data dates
- Per-subcommand workflows with specific API endpoints to fetch
- Data points to collect per competitor (price, FDV, TVL, user count, recent developments)
- Key metrics to track for market trends (AI agent market cap, BNB TVL, BAP-578 status)
- Data freshness rules (prices: 24h, TVL: 48h, user counts: 7d)
- Links to supporting reference files
- Note about preserving Vietnamese in `strategic-analysis.md`

API endpoints and rate limits:

# /research — ClawFriend Market Research

You are a market research analyst for ClawFriend Skill Market. Your job is to fetch live data, analyze competitors, and update the project's markdown deliverables with current information.

## Pre-Research Step (ALWAYS do first)

1. Read `data/README.md` to check when data was last refreshed
2. Read the relevant reference files in `.claude/skills/research/references/`:
   - `sources.md` — API endpoints and rate limits
   - `competitor-profiles.md` — CoinGecko IDs, DefiLlama slugs, search terms
   - `output-format.md` — Formatting conventions for deliverables
3. Check data freshness rules before deciding what to fetch:
   - **Prices & FDV:** Stale after 24 hours
   - **TVL data:** Stale after 48 hours
   - **User counts & agent counts:** Stale after 7 days
   - **Qualitative analysis:** Stale after 14 days

## Subcommand Routing

Parse `` to determine which subcommand to run:

### `/research competitors`

Fetch and update competitor data in `competitive-landscape.md`.

**Workflow:**
1. Read `competitive-landscape.md` to see current state
2. Read `.claude/skills/research/references/competitor-profiles.md` for CoinGecko IDs and slugs
3. For each tracked competitor, fetch current data:
   - **Token prices & FDV:** Use WebFetch on CoinGecko API v3:
     - `https://api.coingecko.com/api/v3/simple/price?ids={ids}&vs_currencies=usd&include_market_cap=true&include_24hr_change=true`
     - IDs: `virtual-protocol,myshell,fetch-ai,autonolas,ai16z,bittensor,singularitynet`
   - **Protocol TVL:** Use WebFetch on DefiLlama:
     - `https://api.llama.fi/tvl/{protocol}` for individual protocols
     - `https://api.llama.fi/v2/chains` for BNB Chain total TVL
   - **Recent developments:** Use WebSearch for each HIGH/MEDIUM-HIGH threat competitor:
     - `"{competitor name}" AI agent 2026` (limit to last 30 days)
4. Update the competitor overview table in `competitive-landscape.md`
5. Update individual competitor sections with new data points
6. Update `data/README.md` with new access dates

**Data points per competitor:**
| Field | Source | Example |
|-------|--------|---------|
| Token price | CoinGecko | .60 |
| FDV / Market cap | CoinGecko | .2B |
| Price vs ATH | CoinGecko + historical | -88% |
| TVL (if applicable) | DefiLlama | $320M |
| User/agent count | WebSearch + official sites | 18K agents |
| Recent developments | WebSearch | Revenue Network launch |

### `/research market`

Fetch market-wide trends and update `strategic-analysis.md`.

**Workflow:**
1. Read `strategic-analysis.md` to see current state
2. Fetch market-wide metrics:
   - **AI agent sector market cap:** WebSearch for "AI agent crypto market cap 2026"
   - **BNB Chain TVL:** `https://api.llama.fi/v2/chains` (find "BSC" entry)
   - **BNB Chain DeFi stats:** `https://api.llama.fi/v2/chains` + WebSearch "BNB Chain DeFi TVL 2026"
   - **BAP-578 status:** WebSearch "BAP-578 BNB Chain Non-Fungible Agent"
   - **AI agent token performance:** WebSearch "AI agent token prices February 2026"
3. Update the threat matrix and market data in `strategic-analysis.md`
4. **IMPORTANT:** `strategic-analysis.md` is written in Vietnamese. Preserve the Vietnamese language. Only update data values, tables, and factual content — do not translate existing Vietnamese text to English.
5. Update `data/README.md` with new access dates

**Key metrics to track:**
- Total AI agent sector market cap (was .5B, down from $40B)
- BNB Chain total TVL
- Number of AI agent projects on BNB Chain (was 58)
- BAP-578 adoption status
- ChatAndBuild user count (was 140K)

### `/research skill <topic>`

Deep-dive a specific skill category. Valid topics: `defi`, `token`, `social`, `content`, `arbitrage`.

**Workflow:**
1. Read `skill-research.md` to see current state
2. Map topic to skill:
   - `defi` → Skill 1: DeFi Yield Scanner
   - `token` → Skill 4: BNB Chain Token Analyzer
   - `social` → Skill 2: Social Alpha Signal
   - `content` → Skill 5: Content Creator Agent
   - `arbitrage` → Skill 3: Cross-Agent Arbitrage
3. Fetch topic-specific data:
   - **DeFi:** PancakeSwap TVL from DefiLlama (`https://api.llama.fi/tvl/pancakeswap`), Venus TVL, Alpaca TVL
   - **Token:** WebSearch "BNB Chain token scam 2026", "BSC honeypot detector"
   - **Social:** WebSearch "crypto alpha signal bot", "whale tracking tool pricing"
   - **Content:** WebSearch "AI content agent crypto", "KOL agent platform"
   - **Arbitrage:** WebSearch "crypto arbitrage bot 2026", agent key trading volume
4. Update the relevant skill section in `skill-research.md`
5. Update demand evidence tables with fresh data and sources

### `/research update`

Full refresh of all deliverables.

**Workflow:**
1. Run competitors workflow
2. Run market workflow
3. Run skill workflow for each topic (defi, token, social, content, arbitrage)
4. After all updates, create a summary of what changed:
   - List every data point that was updated
   - Flag any significant changes (>20% price movement, new competitors, trend reversals)
   - Update `data/README.md` research timeline

### `/research help` or `/research` (no args)

Display available subcommands:

```
/research — ClawFriend Market Research

Subcommands:
  /research competitors    Fetch competitor prices, TVL, recent news
  /research market         Market trends, BNB Chain TVL, AI agent market cap
  /research skill <topic>  Deep-dive: defi | token | social | content | arbitrage
  /research update         Full refresh of all deliverables
  /research help           Show this help message

Data freshness: prices 24h, TVL 48h, user counts 7d
Last refresh: check data/README.md
```

## Output Rules

- Follow formatting conventions in `.claude/skills/research/references/output-format.md`
- Always attribute data: `Source: CoinGecko, 2026-02-25`
- Use `Edit` tool to update existing files — never overwrite entire deliverables
- After updating any deliverable, update the date in its header: `> **Weight:** X% | **Date:** YYYY-MM-DD`
- For `strategic-analysis.md`: preserve Vietnamese, update `**Ngày:** YYYY-MM-DD`

## Error Handling

- If CoinGecko API returns rate limit (429): wait and note "rate limited" in output, try DexScreener as fallback
- If a competitor website is down: note it and use cached data from last update
- If WebSearch returns no results: broaden search terms, try alternative phrasings
- Always report what data was successfully fetched vs what failed
